# tests

This directory contains unit, integration, and end-to-end tests for the zbal engine and API.

## Structure (planned)

- `unit/` — Isolated tests for individual engine modules (signal processing, correlation, attribution, scoring)
- `integration/` — Tests covering multi-module pipelines (e.g., full signal-to-attribution flow)
- `e2e/` — Tests against a running API instance using sample signal payloads
- `fixtures/` — Shared test data (synthetic signals, expected attributions, false-flag scenarios)
